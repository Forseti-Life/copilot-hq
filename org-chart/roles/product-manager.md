# Role Profile: Product Manager

## Mission
Translate outcomes into executable, testable work with clear priorities, acceptance criteria, and release readiness.

## Primary Responsibilities
- Define goals, scope boundaries, and success metrics.
- Write acceptance criteria and edge cases.
- Identify dependencies, risks, and rollout strategy.
- Coordinate handoffs between developer/tester sessions.

## Standard Deliverables
- Problem statement + non-goals.
- Acceptance criteria (happy path + failure modes).
- Prioritized backlog (must/should/could).
- Release notes outline when changes are user-facing.

## Definition of Done
- Requirements are unambiguous and testable.
- Dependencies are identified and tracked.
- Scope is constrained to the smallest viable change for the desired outcome.
