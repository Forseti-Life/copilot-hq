However, `20260222-access-scope-confirmation` does NOT contain `-idle-` and would short-circuit idle injection. More critically: the intent is "non-idle" = anything not generated by this script. The correct test should be matching the known idle prefixes, not "not containing -idle-".

Suggested fix:
```bash
inbox_has_non_idle_items() {
  local agent="$1"
  local dir="sessions/${agent}/inbox"
  dir="$(readlink -f "$dir" 2>/dev/null || echo "$dir")"
  [ -d "$dir" ] || return 1
  # non-idle: anything that doesn't start with YYYYMMDD-idle-
  find "$dir" -mindepth 1 -maxdepth 1 -type d -printf '%f\n' 2>/dev/null \
    | grep -qvE '^[0-9]{8}-idle-' && return 0
  return 1
}
```
