### 1. `find -printf '%f\n'` GNU-only in `inbox_has_non_idle_items` (line 179) â€” CRITICAL
```bash
find "$dir" -mindepth 1 -maxdepth 1 -type d -printf '%f\n' 2>/dev/null \
  | grep -qv -- '-idle-' && return 0
```
On BSD/macOS `find -printf` is not supported and silently produces no output. `grep -qv` on empty input exits 1, so `inbox_has_non_idle_items` always returns false â€” "no non-idle items". Every agent looks idle at all times, causing idle work to be injected over real work on non-Linux. This is the same bug confirmed in 7 other scripts (cycles 11â€“14).

**Minimal diff direction:**
```diff
-  find "$dir" -mindepth 1 -maxdepth 1 -type d -printf '%f\n' 2>/dev/null \
-    | grep -qv -- '-idle-' && return 0
+  while IFS= read -r item; do
+    [[ "$item" != *-idle-* ]] && return 0
+  done < <(ls -1 "$dir" 2>/dev/null)
+  return 1
```
Or: `find "$dir" -mindepth 1 -maxdepth 1 -type d | xargs -I{} basename {} | grep -qv -- '-idle-'`
