## GAP-2077-N — CharacterCalculator unknown rank returns level instead of 0

**File**: `/var/www/html/dungeoncrawler/web/modules/custom/dungeoncrawler_content/src/Service/CharacterCalculator.php` line 174

**Current code**:
```php
return ($rankBonuses[strtolower($rank)] ?? 0) + max(0, $level);
```

**Bug**: For an unknown rank (not untrained/trained/expert/master/legendary), the null coalesce returns 0 but then `max(0, $level)` still adds the level. So "godlike" at level 5 returns 5 instead of 0.

**Fix**: Add an early return for unknown ranks (treat as untrained):
```php
if (!isset($rankBonuses[strtolower($rank)])) {
  return 0;
}
```
Or change the null coalesce to `return ($rankBonuses[strtolower($rank)] ?? null) !== null ? $rankBonuses[strtolower($rank)] + max(0, $level) : 0;`

**Severity**: Low. The only ranks generated by the game engine are the 5 named values (untrained/trained/expert/master/legendary). This code path is unreachable in normal gameplay. However it is a correctness defect and should be fixed to prevent silent data corruption if new rank strings are introduced.

**Disposition**: APPROVE overall reqs 2071-2082 with GAP. File defect ticket for dev.
