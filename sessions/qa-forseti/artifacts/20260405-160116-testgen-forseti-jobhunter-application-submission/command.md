## Happy Path

- [ ] `[TEST-ONLY]` `WorkdayWizardService::advanceStep()` with a valid job context and mock Playwright runner returns a success result without PHP error or exception.
- [ ] `[TEST-ONLY]` `WorkdayWizardService::advanceWizardAutoSingleSession()` advances through all available steps in sequence without throwing; each step writes a log record with correct `job_seeker_id`.
- [ ] `[TEST-ONLY]` Credential retrieval in Playwright context: `WorkdayPlaywrightRunner` receives the expected credential payload from the credential store (unit test with mock store).
- [ ] `[TEST-ONLY]` All 5 `application_submission_*` routes registered by WorkdayWizardService return HTTP 200 for an authenticated user with `access job hunter` permission.

## Edge Cases

- [ ] `[TEST-ONLY]` `advanceWizardAutoSingleSession()` handles a Playwright timeout/non-zero exit by logging a failure record and returning a structured error (no silent failure or PHP fatal).
- [ ] `[TEST-ONLY]` Step advancement with missing or expired credentials returns a user-facing error message and does not proceed to external ATS.
- [ ] `[TEST-ONLY]` Duplicate session invocation (calling `advanceWizardAutoSingleSession()` twice for same job) is handled without duplicate submission records.

## Failure Modes

- [ ] `[TEST-ONLY]` When external Workday ATS is unavailable (mock 503): `WorkdayPlaywrightRunner` returns a structured failure; wizard logs the failure with ATS-unavailable status code; no exception propagates to user-facing layer.
- [ ] `[TEST-ONLY]` Invalid job context (null job_id) passed to `advanceStep()`: method returns a structured error (not PHP exception).

## Permissions / Access Control

- [ ] All 5 `application_submission_*` routes: authenticated user with `access job hunter` → 200.
- [ ] All 5 routes: anonymous user → 403.
- [ ] Admin behavior: same as authenticated (no special admin privilege required).

## Data Integrity

- [ ] Each `advanceStep()` call writes exactly one log record per step; no orphaned records on rollback.
- [ ] Rollback path: disabling `WorkdayWizardService` (comment out service call) restores prior browser-automation behavior without schema changes.
