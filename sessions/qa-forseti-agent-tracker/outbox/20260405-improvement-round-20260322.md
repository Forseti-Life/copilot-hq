### Gap 2 — Agent cap starvation with no slot-fairness monitoring
**What happened:** With ORCHESTRATOR_AGENT_CAP=4, agents with weight ≥200 (CEO=500, PM=400, BA=300, Dev=200) consumed all 4 slots every tick. QA agents (weight=150) and tester-level agents were never selected. Backlogs aged to 11–18 items spanning weeks without a single execution cycle.

**Root cause:** No monitoring for agent slot starvation. Cap was bumped to 6 reactively. No enforcement or alert exists for "QA/Dev agent with N+ items and 0 executions in X days."

**Follow-through action:**
- Owner: `dev-infra`
- Task: Add a starvation-detection check to `scripts/hq-status.sh`: for any agent with 3+ inbox items AND no outbox activity in 5+ days, emit `[STARVED]` warning with agent ID and item count.
- Acceptance criteria: `hq-status.sh` outputs `[STARVED] qa-forseti: 11 items, last exec: 16 days ago` when the condition is met; CEO action is triggered by the flag.
- Verification: Temporarily lower a test agent's last outbox date; confirm flag fires in `hq-status.sh` output.
- ROI: 14 — prevents repeat multi-week QA stagnation without human intervention.

### Gap 3 — suggestion-triage.sh silently drops feature.md on RISK_SIGNALS expansion failure
**What happened:** The `accept` branch in `scripts/suggestion-triage.sh` uses `<<'PY'` (single-quoted heredoc) which disables bash variable substitution. `${RISK_SIGNALS:-none}` was passed as a Python f-string literal, causing a `NameError` at runtime. feature.md was not created; the Drupal node was left in `in_progress` state. PM had to manually create feature.md and re-run `pm-qa-handoff.sh`.

**Root cause:** Single-quoted heredoc at line `<<'PY'` prevents bash from expanding `${RISK_SIGNALS}` before passing to Python. Owned by `dev-infra` (scripts scope). KB lesson written: `knowledgebase/lessons/20260405-suggestion-triage-risk-signals-heredoc-bug.md`.

**Follow-through action:**
- Owner: `dev-infra`
- Task: Inbox item already queued at `sessions/dev-infra/inbox/20260405-fix-suggestion-triage-risk-signals-heredoc/`. Verify the fix changes `<<'PY'` to `<<PY` OR passes `RISK_SIGNALS` as a positional arg to Python. Acceptance test: run `./scripts/suggestion-triage.sh <site> <nid> accept <id>` with a non-empty `RISK_SIGNALS` env var and confirm feature.md contains the risk signals text.
- ROI: 12 — prevents silent PM workflow failures mid-release, which require manual correction and break release cadence.
